# WEB-10 · Ante Bet 开关 UI（2026-07-14）

分支 `web-10-ante-bet`，worktree 隔离实施。

## 目标

把 ENG-6b 修好的 Ante 数学露给玩家：多付 25%，换更高的免费旋转触发率。按**概率诚实原则**，展示的必须是真数字，且不能随配置变化而变成谎言。

## 关键决策：倍速由服务端解析计算，前端不写死

原需求描述里写的是"骰子出现率翻倍""快 1.67 倍"——这些是拍脑袋的数。真实值取决于骰子权重与 `anteScatterFactor`，而管理后台**可以改这些权重**。任何硬编码在前端的数字，一次配置发布之后就会变成对玩家的欺骗。

所以新增 `packages/engine/src/trigger.ts`，用二项分布解析算触发率（无采样噪声）：

- `freeSpinTriggerRate(config, anteEnabled)` — P(X ≥ 4)，X ~ B(30, p_骰子)
- `anteSpeedup(config)` — 两者之比

`/api/config` 随配置一起下发 `anteRule: { costMultiplier, triggerRate, anteTriggerRate, speedup }`，前端只负责渲染。当前配置实测：**1/153 → 1/93，快 1.64×**。

顺带发现并订正了 ENG-6b 报告里的一处**我自己的错误**：曾写"权重 ×2 触发率涨 17 倍"，那是把模拟器输出的 `1/17`（一个比率）误读成了倍数。真实值是 **8.8 倍**（1/152 → 1/17），解析计算与实测互相印证。已在 `CT-1`、`docs/reports/eng6b-ante-math.md`（附勘误说明）、`CLAUDE.md` 三处更正。

## 实施

| 层 | 改动 |
| --- | --- |
| engine | `trigger.ts`（解析触发率 + 倍速），导出到 `index.ts` |
| server | `/api/config` 下发 `anteRule`；spin 扣款按 ante 后的价格做余额校验；免费旋转期间强制忽略 ante |
| web | 盘面上方加注条（勾选框 + 实扣金额 + 倍速 + 触发率明细）；下注区实扣提示；规则页「加注（Ante Bet）」章节；状态存 localStorage |

## 验证

- TDD：`packages/engine/test/trigger-rate.test.ts` 8 项（含"权重 ×2 → 触发率 8~10 倍"的四次方效应断言）、`apps/server/test/ante.test.ts` 7 项。先看它们失败，再写实现。
- 单测：engine 43 / server 40 全绿。
- **Playwright 实测**（`npm run e2e:ante`，16 项全过）：默认关闭 → 点击开启 → 刷新后状态保持 → 开启时余额按 125 扣、服务端 `totalBet` 记 125 → 关闭后恢复 100 → 规则页公示的倍速/触发率与 `/api/config` 逐字段一致 → 刷到免费旋转后加注按钮禁用 → 无 JS 异常。
- 其余三套 e2e（smoke / features / stats）无回归。

### 踩到的坑

e2e 第一版用 100 注去刷免费旋转，400 局全部失败——加注实扣 125，触发率 1/93 意味着要转几百局，余额早烧干了，后面的 spin 全是"余额不足"。**这不是产品的问题，是测试脚本的问题**：改用最小注 10 刷，115 局就触发了。如果当时把这个失败当成产品 bug 去查，会浪费很多时间。
