# WEB-14 · 赢奖历史（最近 N 局，可展开看盘面）

分支：`web-14-history`（基于 origin/main = 2929ce0）

## 需求
玩家在「战绩」里看到最近若干局的 注/赢/倍数/时间，点某条展开该局**终盘盘面**；游标分页翻更早的。数据一律来自服务端，engine 是唯一真相源。

## 契约先行（CT-2 v0.3）
新增玩家侧端点 `GET /api/history?before=<spinId>&limit=20`：
- 需玩家鉴权（未登录 401）；只返回自己的记录，`spins.id` 降序。
- `limit` 默认 20、上限 50；`before` = 游标（上一页最后一条 spinId），返回 `id < before`。
- `nextCursor`：本页最后一条 spinId；本页返回数 < limit（到底）时为 `null`。
- 每条 `HistoryRow`：`spinId, createdAt, mode, isFree(=mode==='free'), bet, totalCost, totalWin, winX(=totalWin/bet), winTier, finalGrid`。
- **免费局口径**与 `/api/stats` 一致：`isFree` 且 `totalCost=0`（不计入投入），`bet` 为锁定注。
- `finalGrid` = 落库 `result_json` **最后一个 cascade 的 gridAfter**（6×5 终盘），前端直接 `board.setGrid` / 小盘面渲染，无需再拉完整 SpinResult。

## 实现
- **服务端** `apps/server/src/app.ts`：新增 `GET /api/history`。SQL 按 `player_id` + 可选 `id < before` 降序取 `limit` 行，JS 解析每行 result_json 抽 finalGrid，合成 HistoryRow。`nextCursor = rows.length === limit ? 末条 id : null`。
- **前端** `apps/web/src/api.ts`：`fetchHistory(before?, limit=20)` + `HistoryRow` 类型。
- **前端** `apps/web/src/main.ts`：战绩面板内新增「最近记录」列表；`loadHistory(reset)` 首开加载 + 「加载更多」翻页（游标）；点某条 `toggleHistoryRow` 展开只读小盘面（`miniBoardHtml`，cell 带 `data-sym` 供 e2e 逻辑比对）。免费局显示「免费」标签。
- **前端** `apps/web/index.html`：战绩卡内加 `#history / #history-list / #history-more`。
- **前端** `apps/web/src/style.css`：历史列表、免费标签、只读 mini-board、加载更多按钮样式。

  - 覆盖：未登录 401、新玩家空+nextCursor null、降序、字段与 spins 表逐条对账（含 finalGrid = 末 cascade gridAfter）、finalGrid 6×5、默认 limit 20、limit 上限 50、游标翻页无重叠、最后一页 nextCursor null、免费局 isFree+totalCost=0、不串号。

## 替用户默认的决策（待确认）
- **finalGrid 内联进列表**（而非另起 detail 端点惰性拉取）：单端点、契约「数据一律来自 /api/history」最直白，20 局 × 30 cell JSON 体积可忽略。
- **展开用只读 mini-board（CSS grid + 牌面字）**，不复用主 Canvas（主画布在游戏区，弹层里复用会打架）。满足契约「或一个只读小盘面」。
- **winX 字段名**（= totalWin/bet，与 admin SpinRow 口径一致）承载「倍数」；<0.05 显示「—」。
- 免费局仍在历史中展示（标「免费」、投入 0），与个人统计口径一致。
