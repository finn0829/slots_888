## 调参过程（权重从初猜到定稿）

命中率靠低价牌权重压制（60→31），触发率靠 scatter 权重（1.6→6），RTP 靠 payoutScale（1.0→0.42，三轮二分逼近：0.408→88.5%，0.445→102.1%，0.425→97.8%，0.42→95.6%）。教训：大奖长尾使单次测量噪声 ±1~2%（10 万次样本毫无意义，校准至少 200 万次）。

## 发现的设计问题（已修/待办）

1. **【已修】ante"骰子权重翻倍"数学不成立**：触发概率 ∝ 权重⁴，权重 ×2 → 触发率 ×17（1/151→1/17），RTP 302%。改为权重 ×1.12 ≈ 触发率 ×1.45（1/104）。契约 CT-1 已更新 v0.2。
2. **【待办 ENG-6】ante 档 RTP 89.2% 低于基础档**——与"Ante 档应略高"的激励设计相反。V1 前端不露出 ante；精调时把系数提到 ~1.15 再验。
3. **【待定】封顶语义**：当前 5000× 是单 spin 封顶，免费旋转整段总和可超（实测最高 7028×）。若要整段封顶需 server 在 feature 层截断，或 engine 增加跨 spin 上限参数。
4. **保底使免费旋转综合频率 ~1/70**（自然 1/151 + 保底 1/132）。这是"满 100 骰子必得"的直接后果，属产品决策已确认；RTP 已把它计入。
