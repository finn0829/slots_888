# 麻将 Slots — 需求池（按工作流切分 · 流内按 ROI 排序）

> 来源：2026-07-14 头脑风暴（三轮：玩法 → 平台 → 视觉/协作方式）。
> 定位：**可真实部署的完整产品**（虚拟币）：服务端权威判定 + 后台运营管理 + 可审计。
> 协作方式：按工作流切分，**多 agent 并行开发**；跨端契约先行，契约变更必须先改 `docs/contracts/` 再改实现。

## 一、已锁定的核心设计

### 玩法

| 项 | 决定 |
|---|---|
| 盘面 | 6 列 × 5 行 = 30 张牌 |
| 中奖判定 | 全盘计数（scatter pays）：同牌 ≥8 张即赔，分 8–9 / 10–11 / 12+ 三档 |
| 核心机制 | 消除下落（tumble）：中奖牌"打出去" → 上方牌落下"摸进来" → 可连锁 |
| 符号 | 中(顶) · 發(高) · 東南西北(中) · 萬筒條(低) · 白板=Wild · 骰子=Scatter · 金牌=倍数(仅免费旋转) |
| 赢奖分级 | 碰(5×) → 杠(10×) → 胡了！(25×) → 自摸(50×) → 天胡(100×+) |
| 连锁倍数 | 基础局 ×1→×2→×3→×5→×10；免费旋转整局累加不重置 |
| 数学目标 | 默认 RTP 96.5%（另 92/94.5/97.5 三档预设）· 高波动 · 命中率~30% · 免费旋转~1/200 · 封顶 5000× |
| **概率诚实原则** | **禁止隐藏动态 RTP**。连败留存一律用透明机制：Ante Bet、骰子收集保底、经济缓冲、真实近失演出 |

### 平台与架构

| 项 | 决定 |
|---|---|
| 判定归属 | **服务端权威**：前端点 spin → 服务端 RNG+engine 结算+存库 → 返回 SpinResult，前端只演出 |
| RTP 调节 | 后台预设档位一键切换 + 高级模式调权重/赔付 + 服务端模拟器实时估算 + 配置版本发布/回滚 |
| 数据库 | SQLite。表：players / spins（全量审计）/ game_configs（版本化）/ transactions |
| 账号 | 游客一键开玩（token 存浏览器），后台管理员密码登录 |
| 代码结构 | monorepo：`packages/engine`（纯 TS 零依赖）· `apps/server`（Fastify + SQLite）· `apps/web`（原生 TS + Canvas + Vite）· `apps/admin`（Vite + React） |
| 运行方法 | 开发 `npm run dev` 一键起三端；**不做线上部署**——本地运行，用 Playwright（已装 Chromium）做体验与端到端验证 |
| 经济数值 | 初始余额 10,000 文 · 注档 [10,20,50,100,200,500] · 签到 1,000 文/日 · 破产补 2,000 文（余额<10，冷却 4h）· 保底满 100 骰子 → 10 次免费旋转 |
| 代码托管 | GitHub。deploy key 放 repo 内 `.ssh/`（已 gitignore），git 用仓库级 `core.sshCommand` 指向它，不碰 `~/.ssh/config` |

### 视觉（雅致国风 —— 茶楼雅室，不是赌场）

| 项 | 决定 |
|---|---|
| 色板 | 深墨绿 `#1B3B2F` 牌桌/主背景 · 象牙白 `#F5EFE0` 牌面 · 烫金 `#C9A45C` 边框/金额 · 朱砂红 `#B33A3A` 中/萬/胡牌横幅 · 竹绿 `#3E7C59` 發/條 |
| 气质 | 牌面书法字（楷体系）· 背景淡墨山水/暗纹 · 赢奖演出金箔粒子+红绸带 · 「胡了！」大字书法压屏。参照雀魂的质感方向但更传统 |
| 工作流 | 先 HTML/SVG 做牌面画廊与 UI mockup（浏览器迭代）→ 定稿后烧成 Canvas 贴图/绘制代码。设计产出放本地 `docs/design/`，不同步 claude.ai |
| 音效 | Web Audio 合成（洗牌/落牌"啪"/各档赢奖），可静音 |

## 二、跨端契约（并行开发的前提，放 `docs/contracts/`）

| # | 契约 | 内容 | 生产者 → 消费者 |
|---|---|---|---|
| CT-1 | `SpinResult` JSON Schema | 每次连锁的盘面快照、消除位置、倍数、各档赢奖、免费旋转状态、保底进度 | engine → server / web / admin(回放) |
| CT-2 | HTTP API 契约 | 玩家侧：建游客会话、spin、查余额、签到；管理侧：登录、配置 CRUD+发布/回滚、模拟器估算、统计聚合、玩家管理、审计查询 | server → web / admin |
| CT-3 | DB Schema | players / spins / game_configs / transactions 建表语句与索引 | server 内部（admin 只经 API） |
| CT-4 | 设计 Token | 色板、字体、间距、牌面尺寸比例、五档演出规格 | design → web / admin |

## 三、需求池（按工作流）

### 工作流 ENG — 引擎与数学（`packages/engine`，纯 TS，可立即开工）

| # | 需求 | 说明 | 优先级 | 验收标准 |
|---|---|---|---|---|
| ENG-1 | 数学模型 + 蒙特卡洛模拟器 | config（符号/赔付/权重）+ simulate；调出 4 套 RTP 档位 | P0 | 每档实测 RTP ≤±0.3%，默认档命中率 28–32%，输出波动率/最大赢分布报告 |
| ENG-2 | Spin 纯函数引擎 | rng（可注入 seed）+ spin：(seed, bet, config) → SpinResult（CT-1） | P0 | 单测覆盖判定/消除/下落/连锁倍数；同 seed 同 config 可复现 |
| ENG-3 | Wild 白板 + 连锁倍数 | 白板并入最优组合；×1→×2→×3→×5→×10 | P1 | 单测 + 模拟器验证不破坏 RTP |
| ENG-4 | 骰子 + 免费旋转数学 | ≥4 触发 10 次，倍数整局累加，再触发规则 | P1 | 触发率 ~1/200 进模拟报告 |
| ENG-5 | 金色倍数牌 | 仅免费旋转，×2/×5/×20/×100 相加后乘总赢 | P2 | 各档 RTP 含金牌仍达标 |
| ENG-6 | Ante Bet 数学 | 注 ×1.25 换骰子率翻倍（~1/100） | P2 | 两模式各自过模拟器，Ante 档 RTP 略高 |
| ENG-7 | 骰子收集保底参数 | 每骰子 +1，满 100 必得免费旋转 | P2 | 保底计入 RTP 模型不破坏总目标 |

### 工作流 SRV — 后端（`apps/server`，依赖 ENG-2 + CT 契约）

| # | 需求 | 说明 | 优先级 | 验收标准 |
|---|---|---|---|---|
| SRV-1 | 脚手架 + DB + 游客会话 | Fastify + SQLite（CT-3）+ 游客建号/token | P0 | 建号→查余额跑通 |
| SRV-2 | Spin API 全流程 | 扣注→engine→落库（seed+配置版本）→返 SpinResult | P0 | 余额不可能为负；spin 全量可审计 |
| SRV-3 | 免费旋转/保底状态持久化 | 免费旋转剩余次数、累计倍数、骰子进度存服务端 | P1 | 刷新页面不丢局 |
| SRV-4 | 配置版本管理 + 模拟器 API | 档位切换、权重编辑、调模拟器估算、发布/回滚，全程留痕 | P1 | 发布后新 spin 立即用新版本；可回滚 |
| SRV-5 | 统计聚合 API | 实测 RTP（按日/按版本）、spin 量、命中率、赢奖分布、免费旋转触发率 | P2 | 与 spins 表逐条可对账 |
| SRV-6 | 玩家管理 + 审计查询 API | 查/补币/重置/封禁；按玩家/时间查任意 spin | P2 | 操作全进 transactions |
| SRV-7 | 经济缓冲 | 每日签到、破产补币（冷却）、连败返利 | P2 | 发币全走流水，参数后台可调 |
| SRV-8 | 管理员鉴权 | 密码登录（env 配置），管理 API 全部鉴权 | P1 | 未登录一律 401 |

### 工作流 WEB — 游戏前端（`apps/web`，依赖 CT-1/2/4）
